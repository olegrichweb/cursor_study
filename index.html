• git config --global user.name blabla задади юсернейм
  • git config --global user.email blabal@bla задати емейл
  • git init - инициализировать репозиторий
  • git status - проерить статус репозитория
  • git add - выбрать файл в репозитории
  • git commit - внести изменения в файл
  • git log - просмотреть историю коммитов
  • git checkout копировать файлы из истории или сцены в рабочую директорию. Также она может использоваться для переключения между ветками.
  • git branch - может использоваться для переключения между ветками.
  • git pull - автоматически получить изменения из удалённой ветки и слить их со своей текущей.
  • git merge - выполнить слияние веток
  • git clone - создать клон копию удаленного репозитория
gitignore нужен для скрытия файлов и папок от системы контроля версий Git. Обычно используеться для скрытия паролей и конфиг файлов

Advanced

  • git revert - отменить последний комит
  • git reset слиянее веток. Подробнее сдесь
  • git rebase - С помощью команды rebase вы можете взять все коммиты из одной ветки и в том же порядке применить их к другой ветке.
  • git cherry-pick - используется для того чтобы взять изменения, внесённые каким-либо коммитом, и попытаться применить их заново в виде нового коммита наверху текущей ветки.
  • git stash - скрывает все сделанные изменения и переводит код в состояние HEAD.
